# @greycat/web вЂ” Development Guide

## Mental Model

`@greycat/web` is **not** React, Vue, or Svelte. Key differences:

- **Web Components** (Shadow DOM + Custom Elements) are the component model. All `gui-*` and `sl-*` elements are standard Custom Elements вЂ” usable from plain HTML, vanilla JS, or JSX.
- **JSX is optional.** It's a thin convenience layer over `document.createElement()`. Everything JSX does, you can do with the DOM API directly. Use whichever style fits the situation.
- There is **no reactivity system**. To update the UI, hold a reference and set properties imperatively.
- **Shoelace** (`sl-*`) is the design system вЂ” all components are pre-registered and fully typed.
- **GreyCat backend** provides typed data via RPC. `gc.sdk.init()` loads the ABI (type metadata) and registers all `gui-*` components.
- Apps are **Multi-Page Applications** (MPA). Each page is a separate HTML entry point with its own script.

## Project Setup

Scaffold a new project with VitePlus+:

```bash
vp create github:datathings/greycat-web-template
cd your-app-name
pnpm run setup
pnpm install
```

`pnpm run setup` fetches the latest GreyCat core and `@greycat/web` versions, updating `project.gcl` and `package.json` automatically.

### What the Template Provides

The scaffolded project uses the `greycat()` VitePlus+ plugin (`vite.config.ts`) which auto-configures:
- `base: './'` вЂ” relative URLs
- `appType: 'mpa'` вЂ” multi-page app
- `root: 'app'` вЂ” VitePlus+ root directory
- `resolve.alias: { '~': 'app' }` вЂ” tilde imports
- Proxies `POST` (RPC) and `/files/*` (GET/PUT/DELETE) to GreyCat backend at `http://127.0.0.1:8080`
- Auto-discovers all `.html` files under `app/` as entry points

Plugin options: `greycat({ greycat: 'http://custom:9090', gzip: true, noDefaultConfig: false })`

### Key Configuration Details

- **`tsconfig.json`**: Uses `"jsx": "react-jsx"` with `"jsxImportSource": "@greycat/web"` for JSX support.
- **`vite.config.ts`**: Imports `defineConfig` from `"vite-plus"` and `greycat` from `@greycat/web/vite-plugin`.
- **`project.d.ts`**: Auto-generated types for `gc.project.*` вЂ” never edit this file. Generated by `greycat codegen ts`.

## Page Entry Pattern

### HTML Shell

Every page is a minimal HTML file that loads a `.ts` or `.tsx` script:

```html
<!doctype html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Dashboard</title>
  </head>
  <body>
    <script type="module" src="./index.tsx"></script>
  </body>
</html>
```

### Entry Script (with JSX)

```tsx
import '@greycat/web';
import '@greycat/web/greycat.css';

await gc.sdk.init({ debug: import.meta.env.DEV });

const data = await gc.project.myFunction();

document.body.appendChild(
  <gui-layout>
    <div slot="header">My App</div>
    <gui-nav slot="navigation" />
    <div slot="main">
      <gui-table value={data} globalFilter />
    </div>
  </gui-layout>,
);
```

### Entry Script (vanilla JS вЂ” no JSX)

```ts
import '@greycat/web';
import '@greycat/web/greycat.css';

await gc.sdk.init({ debug: import.meta.env.DEV });

const data = await gc.project.myFunction();

const header = document.createElement('div');
header.slot = 'header';
header.textContent = 'My App';

const nav = document.createElement('gui-nav');
nav.slot = 'navigation';

const table = document.createElement('gui-table');
table.value = data;
table.globalFilter = true;

const main = document.createElement('div');
main.slot = 'main';
main.appendChild(table);

const layout = document.createElement('gui-layout');
layout.append(header, nav, main);
document.body.appendChild(layout);
```

### Key Points

- `import '@greycat/web'` **must** come first вЂ” it registers all `gui-*` and `sl-*` components.
- `gc.sdk.init()` connects to the backend, loads the ABI, and must complete before using any `gui-*` component or `gc.project.*` call.
- Top-level `await` works because `<script type="module">`.
- Use `greycat.css` for core styles or `greycat-full.css` for classless HTML element styling.
- CSS modules are supported: `import s from './index.module.css'` then `className={s.myClass}`.

## JSX Runtime (Optional)

JSX is a convenience layer, not a requirement.

### Return Type Is a Real DOM Node

```tsx
const div = <div className="foo">Hello</div>;         // HTMLDivElement
const frag = <>item 1, item 2</>;                      // DocumentFragment
const table = (<gui-table value={data} />) as GuiTable; // cast for typed access
```

### Props Mapping

| Prop | Behavior |
|---|---|
| `className` | `string` в†’ `classList.add()`; `string[]` в†’ adds all; `{ name: boolean }` в†’ conditional add/remove |
| `style` | `string` в†’ sets `cssText`; object в†’ assigns each property, `--*` keys use `setProperty()` |
| `$ref` | Callback `(el) => void` invoked at creation |
| `slot` | Sets `element.slot` |
| `on*` | `addEventListener(name.slice(2), handler)` вЂ” e.g. `onclick`, `ongui-change`, `onsl-show` |
| `attr:*` | `setAttribute(name.slice(5), value)` вЂ” for HTML attributes without a DOM property |
| Everything else | Direct property assignment |
| `null` / `undefined` | Silently skipped |

```tsx
<div className={{ active: isActive, hidden: !visible }}>...</div>
<div style={{ display: 'flex', gap: '8px', '--my-color': 'red' }}>...</div>
<gui-chart $ref={(el) => { chartRef = el; }} />
<gui-table ongui-change={(ev) => console.log(ev.detail)} />
```

### `cx()` Utility

```ts
import { cx } from '@greycat/web/jsx-runtime';
cx(element, 'foo');
cx(element, ['foo', 'bar']);
cx(element, { active: true, hidden: false });
```

## Creating Custom Components

### Pattern 1: Plain Function

```tsx
function userCard(name: string, role: string): HTMLElement {
  return (
    <gui-card>
      <div slot="header">{name}</div>
      <p>{role}</p>
    </gui-card>
  ) as HTMLElement;
}
```

### Pattern 2: GuiElement (Shadow DOM)

```tsx
import { GuiElement, css, registerCustomElement } from '@greycat/web';
import style from './my-widget.css?inline';

export class MyWidget extends GuiElement {
  static override styles = [css(style)];

  constructor() {
    super();
    this.shadowRoot.appendChild(<div className="root"><slot /></div>);
  }

  connectedCallback(): void {
    const signal = this.abortSignal();
    window.addEventListener('resize', this._onResize, { signal });
  }

  private _onResize = () => { /* ... */ };
}

registerCustomElement('my-widget', MyWidget);
```

### Pattern 3: GuiValueElement (Reactive Value)

```tsx
import { GuiValueElement, css, registerCustomElement } from '@greycat/web';

export class MyDisplay extends GuiValueElement<string[]> {
  private _value: string[] = [];

  get value(): string[] { return this._value; }
  set value(v: string[]) {
    this._value = v;
    this._internalUpdate();
  }

  override update(): void {
    this.shadowRoot.replaceChildren(<ul>{this._value.map((item) => <li>{item}</li>)}</ul>);
  }
}

registerCustomElement('my-display', MyDisplay);
```

## Built-in Components Reference

### Layout
- `gui-layout` вЂ” Responsive app shell
- `gui-card` вЂ” Card container
- `gui-tabs` / `gui-tab` / `gui-panel` вЂ” Tabbed panels
- `gui-dialog` вЂ” Modal dialog
- `gui-dashboard` вЂ” Widget grid (dockview-based)

### Data Visualization
- `gui-table` вЂ” Data table with sorting, filtering, editing, CSV export
- `gui-chart` вЂ” D3-powered line/bar/area/scatter chart
- `gui-chart2` вЂ” ECharts-powered chart (line, bar, scatter, pie, candlestick, heatmap)
- `gui-donut` вЂ” Pie/donut chart
- `gui-gauge` вЂ” SVG gauge
- `gui-heatmap` вЂ” Heatmap
- `gui-map` вЂ” MapLibre GL map

### Value Display
- `gui-value` вЂ” Formatted display for any GreyCat primitive
- `gui-object` вЂ” Renders object fields as key-value pairs
- `gui-time` вЂ” Formatted time/datetime

### Inputs
- `gui-input` вЂ” Auto-selects input based on GreyCat type
- `gui-input-string` / `gui-input-number` / `gui-input-bool` / `gui-input-time`
- `gui-input-enum` / `gui-input-object` / `gui-input-array` / `gui-input-map`
- `gui-input-geo` / `gui-input-node` / `gui-input-node-time` / `gui-input-node-index`

### Navigation
- `gui-nav` вЂ” Navigation tree (auto-loads `nav.json`)
- `gui-select` вЂ” Custom dropdown with filtering
- `gui-search-input` вЂ” Search field

### Admin
- `gui-users` / `gui-roles` вЂ” User and role management
- `gui-tasks` вЂ” Task list with status tracking
- `gui-files` вЂ” File upload/download manager
- `gui-logs` вЂ” Log viewer

## Gotchas and Pitfalls

1. **No re-rendering** вЂ” hold a reference and mutate: `table.value = newData;`
2. **`gui-*` requires `gc.sdk.init()`** to complete first before use.
3. **Fragment children disappear** when appended вЂ” don't reuse fragments.
4. **Always import `@greycat/web/greycat.css`** for proper styling.
5. **Cast JSX return type**: `const chart = (<gui-chart />) as GuiChart; chart.compute();`
6. **`project.d.ts` is auto-generated** вЂ” never edit. Run `greycat codegen ts` to regenerate.
7. **CSS in Shadow DOM** вЂ” external CSS won't penetrate; use `static styles = [css(inlineStyle)]`.
8. **MPA navigation is full page loads** вЂ” no client-side router; use `<a>` or `window.location`.
